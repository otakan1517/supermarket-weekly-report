# スーパー週次 売上・粗利レポート自動生成（Python × Excel）

POS明細（Excel）を入力として、週次KPI（売上・粗利・粗利率）を集計し、配布用の週次レポート（.xlsx）を**ワンコマンドで自動生成**するツールです。  
出力レポートには **品質チェック（90_quality）** を含め、欠損や不整合などの基本確認を行ったうえで配布できる形にしています。

## 背景（想定する業務課題）
週次レポート作成は、集計（店舗別・部門別・日別推移）に加えて、配布用の体裁調整（¥表示、%表示、列幅など）や入力データの確認が必要で、手作業だと時間がかかりやすくミスも起きやすい業務です。  
また、担当者の手順に依存すると、作成品質のばらつきや引き継ぎコストが発生しやすくなります。

## 目的
- 週次レポート作成の **工数削減**（集計〜整形まで自動化）
- 手順の固定化による **再現性の確保**（ワンコマンド＋設定ファイル運用）
- 品質チェックを同梱し、配布前の確認を標準化することで **信頼性を担保**

## 提供価値（できること）
- **ワンコマンド実行**で週次レポート（.xlsx）を生成（最新週自動 / 指定週）
- KPI（売上・粗利・粗利率）を、用途別（サマリ／店舗別／部門別／日別）に集計
- **配布用の体裁を自動整形**（¥表示＋桁区切り、粗利率0.00%、列幅調整）
- **品質チェック（90_quality）を出力**し、欠損・不整合の有無を確認できる

## 想定利用シーン
- 毎週の定例報告（店舗別・部門別の実績報告）
- 日別推移を見て、曜日/祝日影響の把握
- レポート作成の属人化を避け、引き継ぎしやすい運用へ移行

## 入力 / 出力

### 入力
- POS明細（Excel）
- 祝日マスタ：`data/master/calendar_master.csv`
- 設定ファイル：`configs/config.yml`

※検証用に、ダミーPOSデータ生成スクリプトも用意：`scripts/generate_dummy_pos.py`

### 出力
- `outputs/weekly_report_YYYY-Wxx.xlsx`

出力例（作成・確認済み）
- `outputs/weekly_report_2026-W01.xlsx`
- `outputs/weekly_report_2026-W02.xlsx`

---

## 出力レポートのシート構成（見方）
- `00_readme`：レポートの見方・注意点・配布前チェック
- `01_summary`：週次サマリ（売上・粗利・粗利率の全体値）
- `02_store`：店舗別（売上・粗利・粗利率）
- `03_dept_category`：部門・カテゴリ別（売上・粗利・粗利率）
- `04_trend_daily`：日別推移（曜日/祝日影響の確認）
- `90_quality`：品質チェック（欠損・不整合などの確認）

## 実行方法（ワンコマンド）

### 1) 仮想環境を有効化
```bash
# 例：プロジェクト直下へ移動
cd supermarket-weekly-report

# venv有効化（Mac/Linux）
source .venv/bin/activate
```
### 2) レポート生成（最新週自動）

python scripts/make_weekly_report.py --config configs/config.yml

生成物は outputs/ 配下に出力されます。

非エンジニア向けの詳細手順は [docs/how_to_run.md](docs/how_to_run.md) を参照してください。

週の指定方法（W01/W02の作り分け）

configs/config.yml の target_week_label を変更して実行します。

target_week_label: "2026-W01" → W01生成

target_week_label: "2026-W02" → W02生成

target_week_label: "" → 最新週自動（運用の基本）

設定変更後：

python scripts/make_weekly_report.py --config configs/config.yml

出力の表示整形（配布用フォーマット）

出力レポートは配布を想定し、post_format() で体裁を自動整形します。

金額列：¥表示＋桁区切り

粗利率：%表示（0.00%）

列幅：自動調整（#######回避）

品質チェック（90_quality）

出力レポートには 90_quality シートを含みます。
欠損や不整合などの基本チェック結果を確認でき、問題がなければ INFO ok を表示します。

W01 / W02 ともに INFO ok を確認済み

   工夫した点（実務想定）

ワンコマンド運用：集計〜整形〜品質チェックまでを一括実行し、手順増によるミスを抑制

設定ファイル運用（config.yml）：週指定・列マッピング・入出力をコードから分離し、変更に強い構成

配布用フォーマットの自動整形：¥/%/列幅調整を自動化し、手作業の“仕上げ”を不要に

品質チェック（90_quality）同梱：配布前確認を標準化し、信頼性を担保

再現性のある構成：scripts/configs/data/outputs を責務分離し、引き継ぎ・運用を意識
